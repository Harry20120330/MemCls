# MemCls – Windows 内存清理工具

## 项目概述
MemCls 是一个轻量级的 C# 控制台工具，用于在 Windows 系统上释放内存。它通过一系列低层 Win32/NT API 执行以下操作：

- 清理所有用户进程的工作集（EmptyWorkingSet）。
- 清除待机列表和已修改页列表。
- 刷新系统文件缓存。
- 重新整理注册表缓存。
- 合并物理内存页面。

工具可在普通用户权限下运行（仅执行 `EmptyWorkingSet`），也可以通过 UAC 提升为管理员后执行完整的六项优化。

## 构建配置
项目在 **MemCls.csproj** 中定义了三种发布配置：

| 配置 | 描述 | 产出路径 | 运行时依赖 |
|------|------|----------|------------|
| **JIT**（自包含） | 常规 JIT 编译，发布时捆绑 .NET 运行时，使 exe 可在未安装 .NET 的机器上直接运行。 | `publish/JIT/MemCls.exe`（约 40 MB） | 完全捆绑，无外部依赖 |
| **R2R**（Ready‑to‑Run） | 预编译 + 自包含运行时，启动更快且保持完整 .NET 功能。 | `publish/R2R/MemCls.exe`（约 45 MB） | 完全捆绑 |
| **AOT**（Native AOT） | 完全本机编译，单文件、最小体积。 | `publish/AOT/MemCls.exe`（约 4‑6 MB） | 完全捆绑 |

### 构建示例
```powershell
# JIT（自包含）
 dotnet publish -c JIT -o publish/JIT

# Ready‑to‑Run (R2R)
 dotnet publish -c R2R -o publish/R2R

# Native AOT
 dotnet publish -c AOT -o publish/AOT
```
上述三种方式均会在可执行文件同目录下生成 `log` 文件夹，运行时日志写入其中。控制台仅输出错误信息，其他信息记录到日志文件。

## 使用方法
1. 运行 exe。若当前不是管理员，程序会弹出 UAC 提示请求提升。
2. 选择 **Yes** 进行完整清理（执行全部六项优化）。
3. 选择 **No** 仅执行 `EmptyWorkingSet`（受限清理）。
4. 完成后程序会显示内存前后状态以及释放的内存量。

## 配置
可以通过命令行开关（后续可扩展）或在可执行文件旁放置 `memcls.json` 配置文件来修改行为。

## 日志
所有日志等级（`Info`、`Warning`、`Error`）均写入 `log/memcls_YYYYMMDD.log`，仅 `Error` 会在控制台显示，保持 UI 干净。
