# MemCls – Windows Memory Cleaner

## Overview
MemCls is a lightweight native C# console utility that frees up memory on Windows systems. It uses a collection of low‑level Win32/NT APIs to:
- Empty working sets of all user processes.
- Purge standby and modified page lists.
- Flush the system file cache.
- Reconcile registry cache.
- Combine physical memory pages.

The tool can run with normal user rights (fallback to `EmptyWorkingSet`) or request Administrator privileges via UAC for a full clean.

## Build Configurations
The project ships three publish configurations defined in **MemCls.csproj**:

| Configuration | Description | Output | Runtime Dependency |
|---------------|-------------|--------|--------------------|
| **JIT** (self‑contained) | Regular JIT compilation, but the publish bundles the .NET runtime, so the exe can run on machines without .NET installed. | `publish/JIT/MemCls.exe` (≈ 40 MB) | None – fully bundled |
| **R2R** | Ready‑to‑Run (pre‑compiled) + self‑contained runtime – faster start‑up while keeping full .NET features. | `publish/R2R/MemCls.exe` (≈ 45 MB) | None – fully bundled |
| **AOT** | Native AOT – completely native binary, single file, smallest size. | `publish/AOT/MemCls.exe` (≈ 4‑6 MB) | None – fully bundled |

### Building
```powershell
# JIT (self‑contained)
 dotnet publish -c JIT -o publish/JIT

# Ready‑to‑Run
 dotnet publish -c R2R -o publish/R2R

# Native AOT
 dotnet publish -c AOT -o publish/AOT
```
All three produce a `log` folder next to the exe where runtime logs are written. Console output only shows errors; informational messages go to the log file.

## Usage
1. Run the exe. If you are not an administrator the program will request elevation.
2. Choose **Yes** to run the full clean (all six optimization steps).
3. Choose **No** to run the limited clean (only `EmptyWorkingSet`).
4. After completion the program prints the memory before/after stats and the amount of freed memory.

## Configuration
You can modify the behaviour via command‑line switches (future extension) or by editing the `memcls.json` configuration file placed beside the executable.

## Logging
All log levels (`Info`, `Warning`, `Error`) are written to `log/memcls_YYYYMMDD.log`. Only `Error` messages are echoed to the console, keeping the UI clean.
